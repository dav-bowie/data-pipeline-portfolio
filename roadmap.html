YOUR LEARNING ROADMAP

Four phases. Each builds on the last. Phase 1 is your foundation — don't skip it. By Phase 4, companies will pay for your pipelines.

01
Phase 1 — Foundation (Weeks 1–4)

Databases, SQL & Python Basics

Before you can build pipelines, you need to speak the language of data. SQL is the most in-demand skill in analytics. Python makes you unstoppable. Databases are where everything lives.

  • Set up PostgreSQL locally and learn: CREATE TABLE, INSERT, SELECT, WHERE, JOIN, GROUP BY, ORDER BY
  • Learn Python fundamentals: variables, loops, functions, file I/O — focus on pandas and numpy
  • Connect Python to PostgreSQL using psycopg2 or SQLAlchemy — query and write data programmatically
  • Understand data types, primary keys, foreign keys, and basic indexing
  • Practice: load a CSV into PostgreSQL and query it with SQL, then replicate in Python
  • Resource: "Mode SQL Tutorial" (free), "Kaggle Python Course" (free), PostgreSQL Docs
02
Phase 2 — ETL & Data Cleaning (Weeks 5–8)

Build Your First Pipeline

ETL = Extract, Transform, Load. This is the core of every data job. You'll learn why source data is always messy and how to detect root-cause errors — the skill AI cannot replicate.

  • Learn pandas deeply: merge, groupby, pivot_table, fillna, drop_duplicates, apply, lambda
  • Build data validation functions: check nulls, check referential integrity across linked tables
  • Write a Python ETL script: pull data from a CSV/API → clean it → load to PostgreSQL
  • Use Jupyter Notebooks for exploratory data analysis (EDA) — this is your analysis scratchpad
  • Learn to detect data lineage issues: trace errors from source tables back to root cause
  • Build a root-cause analysis template: when data looks wrong, ask "which upstream table caused this?"
03
Phase 3 — Visualization & Storytelling (Weeks 9–12)

Tell the Story in the Data

Numbers mean nothing without context. Your job as an analyst is to answer: what happened, why, and what should we do? This is where you go from "analyst" to "decision driver."

  • Learn Plotly and Matplotlib for Python charts: bar, line, scatter, heatmap, treemap
  • Build a Streamlit dashboard — run Python code as a live web app with zero frontend knowledge
  • Frame every analysis with: Objective → Key Metric → Root Cause → Recommendation
  • Practice storytelling: given revenue data, find the top 3 insights and explain them in 3 bullets
  • Learn business KPIs: CAC, LTV, churn rate, gross margin, MoM growth, cohort analysis
  • Build a revenue/cost analysis report for a fictional SaaS company using public data
04
Phase 4 — Automation & Pipelines (Weeks 13–20)

N8N, Airflow & Live Systems

This is where you become irreplaceable. You're not just analyzing data — you're building the infrastructure that auto-refreshes, validates, and delivers insights without anyone pressing a button.

  • Install N8N locally, then on a cloud VPS (DigitalOcean/Render) — learn its node-based pipeline builder
  • Build an N8N pipeline: fetch data from an API → transform → insert to PostgreSQL → send Slack alert
  • Schedule pipelines with cron triggers — automate your ETL to run daily/hourly
  • Add error handling nodes: if data is missing or schema changes, send an alert email automatically
  • Connect N8N to your Streamlit dashboard for a fully automated live reporting system
  • Learn Apache Airflow as the enterprise-grade version of N8N for larger orgs
  • Build and document 2–3 sellable pipeline templates and publish on GitHub with a README

SKILLS DEVELOPMENT PLAN

You identified four gaps. Here's exactly how to close each one with free resources and a progression path.

Databases (PostgreSQL)
START HERE
Week 1–2: Install PostgreSQL + pgAdmin. Do the Mode Analytics SQL Tutorial (free, interactive). Create your first database with 3 linked tables.

Week 3–4: Learn JOINs deeply — INNER, LEFT, RIGHT, FULL. Practice with the "DVD Rental" sample database that comes with PostgreSQL.

Milestone: Write a query that joins 3 tables and produces a revenue-by-customer-segment report.
Python for Data
MOST IMPORTANT
Week 1–2: Kaggle's free Python and Pandas courses — hands-on, browser-based, no setup needed.

Week 3–4: Build your own ETL script. Read a CSV, clean it with pandas, write to PostgreSQL.

Week 5+: Learn requests library for API calls, schedule library for simple automation, logging for pipeline monitoring.

Milestone: A Python script that runs ETL on a real dataset end-to-end.
SQL (Advanced)
HIGH DEMAND
Foundation: SELECT, WHERE, GROUP BY, ORDER BY, LIMIT, HAVING

Intermediate: CTEs (WITH clauses), window functions (ROW_NUMBER, LAG, LEAD, RANK), subqueries

Advanced: Query optimization, EXPLAIN ANALYZE, indexing strategies, materialized views

Resource: pgexercises.com (free, in-browser), LeetCode SQL problems (free tier), Mode Analytics SQL School
Analysis & Storytelling
YOUR MOAT
Framework to learn: For every dataset, ask: What changed? When did it change? What caused it? What does it mean?

Practice: Find a public dataset on Kaggle. Write a 1-page analysis in a Jupyter Notebook answering those 4 questions with charts.

Business KPIs to learn: MoM growth, cohort retention, CAC payback period, gross margin, funnel conversion rates

Milestone: Present 3 insights from a dataset to someone non-technical. If they understand it, you've won.
